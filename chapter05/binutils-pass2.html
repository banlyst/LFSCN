5.8. Binutils-2.22 - 第二遍

Binutils 程序包包含一个链接器、一个汇编器和其他用于处理对象文件的工具。

预计编制时间: 1.1 SBU
需求磁盘空间: 363 MB

5.8.1. 安装 Binutils

应用一个补丁防止使用编译器调节时编制失败： 

patch -Np1 -i ../binutils-2.22-build_fix-1.patch

再创建一个独立的编制目录： 

mkdir -v ../binutils-build
cd ../binutils-build

准备编译 Binutils： 

CC="$LFS_TGT-gcc -B/tools/lib/" \
AR=$LFS_TGT-ar                  \
RANLIB=$LFS_TGT-ranlib          \
../binutils-2.22/configure      \
        --prefix=/tools         \
        --disable-nls           \
        --with-lib-path=/tools/lib

新的配置选项的含义：

CC="$LFS_TGT-gcc -B/tools/lib/" AR=$LFS_TGT-ar RANLIB=$LFS_TGT-ranlib

因为这一次是原生编制 Binutils，设置这些变量可以确保编制系统使用交叉编译器和相关工具而不是宿主系统上的工具。

--with-lib-path=/tools/lib

此项告诉配置脚本在 Binutils 编译期间指定库搜索路径，使得 /tools/lib 传递给链接器。这就防止了链接器搜索宿主上的库目录。

编译程序包： 

make

安装程序包： 

make install

现在为了下一章中的 “重新调节” 一节准备链接器： 

make -C ld clean
make -C ld LIB_PATH=/usr/lib:/lib
cp -v ld/ld-new /tools/bin

make 参数的含义：

-C ld clean

此项告诉 make 程序移除 ld 子目录中所有编译产生的文件。

-C ld LIB_PATH=/usr/lib:/lib

此项重新编制 ld 子目录中的一切。在命令行中指定 LIB_PATH Makefile 变量允许我门覆盖临时工具的默认值并指向正确的最终路径。此变量的值指定了链接器默认的库搜索路径。这项准备工作在下一章中用到。

有关此程序包的细节位于第 6.13.2 节 “Binutils 的内容”。
